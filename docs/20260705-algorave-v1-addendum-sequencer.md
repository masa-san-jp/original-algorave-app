# Algorave Terminals v1 — 追補: STEPモード(視覚的ステップシーケンサー)+ ループ進行プレイヘッド

作成日: 2026-07-05
版数: 1.0(機能仕様書 `20260703-algorave-v1-external-spec.md` への追補。既存FRを変更しない加算のみ)

## 背景

v1のコードエディタ(CODEモード)はStrudel/Tidal互換の全記法を表現できる反面、初見のユーザーにはテキスト入力の学習コストがある。「より視覚的・直感的に操作できるシーケンサーモード」と「ターミナル上でループが進行するUI」を追加要件として実装した。

## FR-13: STEPモード(ステップシーケンサー)

1. 各トラックのヘッダに `⊞ STEP` / `⌨ CODE` の切替ボタンを追加する。既定は CODE モード(既存動作は完全に維持)。
2. STEPモードは **BD / SD / HH / OH / CP の5レーン × 16ステップ**の固定グリッドを表示する(1グリッド = 1小節、16分音符相当)。4ステップごとに区切り線を入れ拍を視認できるようにする。
3. セルをクリックすると即座にそのレーン・ステップがトグルされ、**次のサイクル境界から音に反映される**(RUNボタンを待たない)。これは既存の「AIはRUNを待つ(事故防止)」原則とは異なる意図的な設計判断であり、理由は次の通り:
   - STEPモードのセル操作は常に構文的に妥当な状態しか生成しない(パースエラーが原理的に起こり得ない)ため、CODEモードの「編集途中の壊れた構文」という事故リスクが存在しない。
   - 物理的なステップシーケンサー(TR-808等)の操作感は「叩いたら鳴る」が期待値であり、直感性を優先した。
4. **グリッドはコードの別ビュー**であり、独立した状態を持たない。セル操作は内部で `bd ~ ~ bd ...`(16トークン/行)形式のコードを生成し、既存の `parse()` → `runTrack()` パイプラインをそのまま通す(パーサ・スケジューラは無変更)。そのためCODEモードに戻すといつでも等価なコードが編集用テキストとして見える。
5. CODEモード→STEPモードへの切替時、現在の `code` を可能な範囲でグリッドへ変換する(`stepgrid.ts: codeToGrid`)。変換規則: 各行がちょうど16個の空白区切りトークンで、休符 `~` 以外はすべて単一レーン名(別名可: `sn`→SD, `kick`→BD, `clap`→CP)のときのみ、その行を対応レーンへ反映する。1行でもこの形に一致しない場合(グループ・修飾子・音名・16以外の分割数等)は「現在のコードはSTEPモードで完全には表現できません(変換できた範囲のみ反映)」と表示し、変換できなかった分は空のまま新規グリッドとして開始する。
6. STEPモードは全記法のうち「フラットな等分割パターン」のみを表現する部分集合であり、`[ ]` `< >` ユークリッド等の高度な記法はCODEモードでのみ編集可能(既存のまま)。この非対称は意図的なスコープ:STEPモードは入門・直感操作、CODEモードは全記法という役割分担。

## FR-14: ループ進行プレイヘッド

1. **CODEモード**: RUNNING中のトラックは、エディタ領域の左端から右端まで1小節(`(60/BPM)×4`秒)かけて走査する縦の光る線(プレイヘッド)を表示する。既存の枠発光(`beatPulse`)と同じ「`.running`クラス + `var(--measure)`によるCSSアニメーション」の規約に従うため、BPM変更に自動追従する。
2. **STEPモード**: RUNNING中のトラックは、16ステップを離散的に(`steps(16, jump-start)`)チェイスするハイライトを表示し、現在鳴っているステップ位置を直接示す。
3. 両方とも `prefers-reduced-motion: reduce` で無効化される(既存のNFR-07に準拠)。

## スコープ外・既知の制約

- STEPモードの表示/非表示(`mode`)とグリッド内容(`grid`)はlocalStorageに永続化しない。リロード後は常にCODEモードに戻るが、演奏内容そのもの(`code`)は既存のFR-11でそのまま復元される(単に「どちらの見た目で開くか」がリセットされるだけで、音楽的内容は失われない)。
- ステップ数は16固定(v1では可変不可)。8/32等の解像度切替は将来検討。
- STEPモードのセル操作による即時反映(RUNを待たない)は、CODEモードのテキスト編集には一切影響しない(CODEモードは既存どおりRUN必須のまま)。

- `app/src/style.css` にプレイヘッド・ステップグリッドのスタイルを追加。
