# Algorave Terminals — ローカルサーバー起動手順

作成日: 2026-07-03

## 構成
- `index.html` 1ファイルのみ。ビルド不要、npm不要。Tone.jsはCDN(cdnjs)から読み込むためネット接続だけ必要
- チャットのアーティファクト環境の制約(音声・APIアクセス)を受けない独立版

## 1. PCで起動する

`index.html` を任意のフォルダ(例: `algorave/`)に置き、そのフォルダで以下のいずれかを実行。

Python(Mac/Linuxは標準搭載):
```bash
cd algorave
python3 -m http.server 8000
```

Node.jsがある場合:
```bash
npx serve algorave -l 8000
```

ブラウザで http://localhost:8000 を開く → 「▶ タップして演奏開始」でデモが鳴る。

補足: サーバーなしで `index.html` をダブルクリック(file://)でも音は鳴るが、AI機能のAPI通信がブラウザによってブロックされることがあるため、サーバー経由を推奨。

## 2. スマホから使う

1. PCとスマホを同じWi-Fiに接続
2. PCのローカルIPを確認
   - Mac: `ipconfig getifaddr en0`
   - Windows: `ipconfig`(IPv4アドレスの行)
3. スマホのブラウザで `http://<PCのIP>:8000` を開く(例: `http://192.168.1.23:8000`)
4. 幅760px未満では自動的に TRACK A / B のタブ切替UIに変形

うまく開けない場合: PC側ファイアウォールで8000番ポートを許可、またはWi-Fiルーターの「プライバシーセパレーター(端末間通信遮断)」設定を確認。

## 3. AI機能のセットアップ(任意)

1. https://console.anthropic.com でAPIキーを発行(sk-ant-...)
2. 画面上部の「APIキー」欄に貼り付け(ブラウザのlocalStorageに保存され、次回から不要)
3. 「AI指示」欄に方向性を書き、各トラックの ✦AI → 生成コードを確認して RUN

セキュリティ注意: キーはブラウザから直接Anthropic APIへ送信される方式(自分専用のローカル利用を想定)。他人がアクセスできるURLで公開する場合は、キーをサーバー側に置くプロキシ構成に変えること。

## 4. 動作確認チェックリスト

- [ ] PC: タップ後にキック+ハット+ベースのデモが鳴る
- [ ] コード書き換え → Ctrl+Enter で次の小節から変化する
- [ ] VOL / FILT スライダーがリアルタイムに効く
- [ ] スマホ: A/Bタブで2トラックを行き来できる
- [ ] iPhoneで無音の場合: マナーモード(サイレントスイッチ)を解除

## トラブル時の切り分け

| 症状 | 原因候補 | 対処 |
|---|---|---|
| 画面が真っ白 | CDN未到達(オフライン) | ネット接続を確認 |
| タップしても無音 | OS音量 / iOSマナーモード | 音量・スイッチ確認 |
| スマホから開けない | 別Wi-Fi / ファイアウォール | 同一Wi-Fi・ポート許可 |
| AIがエラー | キー未入力 / 残高 / file://起動 | キー確認、http経由で起動 |
