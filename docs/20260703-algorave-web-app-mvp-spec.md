# Algorave Terminals — MVP仕様書

作成日: 2026-07-03

## コンセプト
ブラウザだけで動くライブコーディング音楽アプリ。トラック = ターミナルウィンドウ。デスクトップでは複数ターミナルを並べてミキシング、スマホでは2トラック(A/B)を行き来してパフォーマンスする。

## 技術構成
- React + Tone.js(Web Audio)。インストール不要、ブラウザで完結
- 音源はすべてシンセ生成(キック/スネア/ハイハット/クラップ/ソウシンセ)。サンプル読み込みなし = モバイルでも軽量
- AI: Anthropic API(claude-sonnet-4-6)でパターンを生成・発展

## ミニ記法
TidalCycles / Strudel のミニノーテーションを簡略化した独自DSL。

| 記法 | 意味 |
|---|---|
| `bd` `sn` `hh` `oh` `cp` | キック / スネア / クローズハット / オープンハット / クラップ |
| `c2` `e3` `g#2` | シンセ音(音名+オクターブ) |
| `~` | 休符 |
| `[a b]` | 1ステップを細分化 |
| `tok*n` | ステップ内でn回反復 |
| 改行 | 並列レイヤー(1行 = 1小節) |
| `#` | コメント |

例:
```
bd ~ bd ~
~ sn ~ sn
hh*8
c2 ~ [c2 c3] g1
```

## 操作
- **RUN(Ctrl/⌘+Enter)**: エディタの内容を次サイクルから反映(ライブコーディングの基本動作)
- **トラックごと**: STOP / MUTE / VOL / FILT(ローパス80Hz〜12kHz)
- **マスター**: BPM(80〜180)、START / STOP ALL、+ TRACK(最大6)
- **モバイル(幅760px未満)**: TRACK A / B の大型タブで切替、片方を鳴らしたままもう片方を書き換えるA/B奏法
- **AI**: 画面上部の指示欄に「もっとアシッドに」等を入力 → 各トラックの✦AIボタン → 生成コードがエディタに入る → 目視確認してRUN(自動反映しない設計。事故防止)

## 設計判断
- **Strudelを埋め込まず自作パーサにした理由**: 単一ファイルで完結・CDN依存なし・モバイル軽量。記法は互換方向に寄せてあるため、将来Strudel移行しても演奏感覚は引き継げる
- **AIは提案→人間がRUN**: ライブ中に音が勝手に変わる事故を防ぐ。ステージング方式
- **サイクル同期の発光**: 実行中ターミナルの枠がBPM同期で明滅し、視覚的にビートを共有(algoraveのプロジェクション文化を意識)

## 既知の制約(MVP)
- 音色は固定5種+ソウシンセのみ(サンプル・音色指定なし)
- エフェクトはローパスのみ(ディレイ/リバーブ未実装)
- スイング、ユークリッドリズム `(3,8)`、速度変更 `/2` 未対応
