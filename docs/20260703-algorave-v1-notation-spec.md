# Algorave Terminals v1 — 記法仕様書(ミニノーテーション)

作成日: 2026-07-03
版数: 1.0

本書は Algorave Terminals v1 のパターン記法を、実装者間で解釈が割れない粒度で定義する。§8 のゴールデンテストベクタは単体テストとして**全件実装・全件パスが検収条件**である。

## 1. 互換方針

TidalCycles / Strudel の「ミニノーテーション」のサブセットを採用する。目標は「Strudel のミニノーテーション文字列(`s("...")` の中身)を書き慣れた人が、読み替えなしでそのまま打てる」こと。

- 採用する要素: 空白区切りシーケンス、`~`、`[ ]`、`< >`、`,`、`*` `/` `!` `@` `?`、ユークリッド `(k,n[,r])`
- 採用しない要素と代替案内は §9 に列挙する(未対応記法は**エラーとして明示的に通知**し、黙って無視しない)。

## 2. 字句(トークン)

- 文字コード: UTF-8。パース対象は ASCII を原則とする。ただし全角スペース(U+3000)は**字句解析の最初の正規化ステップで半角スペースへ変換してから**ASCII前提のパースに進むこと(スマホIMEからの入力対策)。この正規化より後の処理はASCIIのみを扱う想定でよい。
- 行区切り: `\n`(`\r` は除去)。**1行 = 1レイヤー = 1サイクル(1小節)**。複数行は同時再生。
- コメント: **行頭または空白の直後の `#`** から行末まで。コメント除去後に空白のみの行は無視。
  - ⚠️ シャープ音名(`g#2`)の `#` はトークン内文字でありコメントではない。「`#` の直前が行頭か空白のときのみコメント開始」という規則で両立させること(参照プロトタイプは無条件に `#` 以降を除去するため `g#2` が無音になるバグがある。v1 では本規則で修正する)。
- 区切り: 半角スペース(連続スペースは1つと同じ)。タブはスペースとして扱う。
- 大文字小文字: 区別しない(パース時に小文字化)。`BD` = `bd`。

### 2.1 サウンドトークン

| トークン | 別名 | 音 |
|---|---|---|
| `bd` | `kick` | キック |
| `sd` | `sn` | スネア |
| `hh` | — | クローズドハイハット |
| `oh` | — | オープンハイハット |
| `cp` | `clap` | クラップ |

※ 正式名は TidalCycles のサンプル名に合わせ `bd` `sd` を採用。`sn` `kick` `clap` は移行互換の別名。

### 2.2 音名トークン

正規表現: `^[a-g](#|b)?[0-8]$`

例: `c2` `e3` `g#2` `eb2` `a#1`。シンセ(ソウ波)で発音される。

### 2.3 休符

`~` のみ(Tidal と同じ)。

## 3. 文法(EBNF)

```ebnf
line      = stack ;
stack     = sequence { "," sequence } ;          (* コンマ = 同スパン内の同時再生 *)
sequence  = element { WS element } ;             (* WS = 1個以上の空白。§2の正規化・字句分割で要素はすでに空白除去済みのトークン列になっているため、実装上は「配列の要素を順に処理する」形で自然にこの規則を満たす *)
element   = atom { modifier } ;
atom      = token | rest | group | alt ;
group     = "[" stack "]" ;
alt       = "<" sequence ">" ;                   (* サイクルごとに交替 *)
rest      = "~" ;
modifier  = "*" posInt                           (* 高速化(反復) *)
          | "/" posInt                           (* 減速(nサイクルに1回) *)
          | "!" posInt                           (* シーケンス上に複製 *)
          | "@" posInt                           (* ステップ幅の重み *)
          | "?" [ prob ]                         (* 確率トリガー *)
          | "(" posInt "," posInt [ "," nonNegInt ] ")" ; (* ユークリッド (k,n[,r]) *)
token     = drum | note ;
posInt    = 正の整数(1以上) ;
nonNegInt = 0以上の整数 ;
prob      = 0より大きく1以下の小数 (例 0.3) ;
```

※ 数値の**範囲**(`*n` は1〜64、`(k,n,r)` の r は 0 ≦ r < n 等)は構文(EBNF)のレベルでは規定しない。EBNF は `posInt`/`nonNegInt` という整数の符号のみを区別し、実際の許容範囲は次表の意味検査(E-04)で行う。これにより「`int` は正の整数」という構文規則と「`r` に 0 を許可する」という意味規則が矛盾しない。

数値の許容範囲(範囲外は E-04):

| 記法 | 範囲 |
|---|---|
| `*n` | 1〜64 の整数 |
| `/n` | 2〜16 の整数 |
| `!n` | 2〜16 の整数 |
| `@n` | 1〜16 の整数 |
| `?p` | 0 < p ≦ 1(省略時 0.5) |
| `(k,n,r)` | 1 ≦ k ≦ n ≦ 64、0 ≦ r < n |

修飾子の合成規則:

- 修飾子は atom の直後に空白なしで連続でき、**左から右へ順に適用**する(例 `hh*4?0.3` = 4連打してから各イベントを確率30%に)。
- `!n` と `@n` はレイアウト修飾子であり、**連鎖の最後にのみ**置ける(`bd!2*2` は E-05)。`!` と `@` の併用も E-05。
- `/n` は**単一トークンにのみ**適用可(グループ・交替に付けると E-06)。`[bd sd]/2` 相当は `<bd sd>` を案内する。
- 同種修飾子の重複(`bd*2*2` 等)は E-05。

## 4. 意味論(タイミングモデル)

- 各行は独立したパターン。パターンは `query(c: サイクル番号) → Event[]` として評価される。
- `Event = { pos, dur, tok }`。`pos` はサイクル内位置(0 ≦ pos < 1)、`dur` はサイクル比の長さ。位置・長さは**有理数演算または倍精度で誤差 1e-9 未満**を保証すること(`1/3` 等の分割があるため、テストは許容誤差 1e-9 で比較する)。
- **シーケンス**: スパン(親から与えられた区間、行なら [0,1))を各要素の重み合計で比例配分する。重みは既定1、`@n` で n、`!n` は重み1×n個への複製。
- **`[ ]` グループ**: 1要素として1ステップを占め、その内部で再帰的に同じ規則を適用。
- **`,` スタック**: 同一スパンに複数シーケンスを重ねる(イベントは和集合)。行トップレベルと `[ ]` 内の両方で使用可。
- **`*n`**: 要素の内容をそのステップ内で n 回繰り返す(各繰り返しはステップの 1/n 幅)。
- **`/n`**: そのトークンを `c mod n == 0` のサイクルのみ発音する(位置は不変)。※ Strudel の slow はパターンを引き伸ばすが、v1 は単一トークン限定の簡易意味論とする(体感互換)。
- **`< >` 交替**: 中身の要素列を `要素[c mod 要素数]` で1つ選ぶ。選ばれた要素がグループ等なら再帰評価する。ネスト時の `c` は常にグローバルなサイクル番号。
- **`?p`**: その atom が生成した各イベントを、イベントごと・サイクルごとに独立の確率 p で残す。乱数は非シードの一様乱数(`Math.random`)でよい。ただしテスト容易性のため乱数関数は注入可能にすること(§8 の確率系テストはモック乱数で検証する)。
- **`(k,n,r)` ユークリッド**: そのステップを n 等分し、Bjorklund アルゴリズム(§5)による E(k,n) のオンセット位置でのみ発音する。各発音の dur は ステップ幅/n。r は左回転(§5.2)。

## 5. ユークリッドリズム

### 5.1 Bjorklund アルゴリズム(規範実装)

```
function bjorklund(k, n):            # k個の1をn個のスロットへ最も均等に配置
  a = [[1]] * k                      # 1のグループ
  b = [[0]] * (n - k)                # 0のグループ
  while len(b) > 1:
    pairs = min(len(a), len(b))
    newA = [a[i] + b[i] for i in range(pairs)]
    rest = a[pairs:] if len(a) > pairs else b[pairs:]
    a, b = newA, rest
  return flatten(a + b)              # 1/0 の配列(長さn)
```

規範ベクタ(実装が正しいことの必要条件):

| (k,n) | ビット列 | オンセット index |
|---|---|---|
| (1,4) | `1000` | 0 |
| (2,4) | `1010` | 0,2 |
| (3,8) | `10010010` | 0,3,6 |
| (5,8) | `10110110` | 0,2,3,5,6 |
| (7,16) | 実装のBjorklund出力に従う(テストではビット列でなくオンセット数=7と均等性のみ検証) | — |

### 5.2 回転 `r`

左回転: `rotated[i] = original[(i + r) mod n]`。

例: E(3,8) = `10010010`、r=1 → `00100101`(オンセット index 2, 5, 7)。

## 6. 乱数(`?`)

- イベント単位・サイクル単位で独立試行。
- パーサ/パターンモジュールは乱数関数 `rng(): number`(0以上1未満)を外部注入できる構造にする。プロダクションでは `Math.random`。
- 判定: `rng() < p` のとき発音。

## 7. エラー仕様

パースエラーはコード(E-xx)と固定文言で通知する。`{}` は埋め込み変数。行番号は1始まり。

| コード | 条件 | 文言 |
|---|---|---|
| E-01 | 括弧の不一致(`[` `<` `(` が閉じない/対応しない閉じ括弧) | `記法エラー(行{n}): "{括弧}" が閉じられていません` / `記法エラー(行{n}): 対応しない "{括弧}" があります` |
| E-02 | 不明なトークン | `不明なトークン(行{n}): "{tok}"` |
| E-03 | 未対応の記法文字(`{` `}` `%` `:` `.` `"` `'` `\|` `+` `-` 等、文法外の文字) | `未対応の記法(行{n}): "{文字}" — 対応記法はヘルプ参照` |
| E-04 | 数値が範囲外・不正(`*0` `?1.5` `(9,8)` 等) | `数値が不正です(行{n}): "{記述}"` |
| E-05 | 修飾子の不正な組合せ(§3) | `修飾子の組合せが不正です(行{n}): "{記述}"` |
| E-06 | `/n` をグループ・交替に適用 | `"/{n}" は単一トークンにのみ使えます(行{n})。<a b> を検討してください` |

- 複数行に複数エラーがある場合、**最初のエラー1件**を表示すれば足りる。
- エラー時の RUN の挙動(旧パターン継続)は機能仕様書 FR-03 に従う。

## 8. ゴールデンテストベクタ

- 位置・長さは分数で表記(実装テストでは誤差 1e-9 未満で比較)。
- `c` はサイクル番号。`c` の記載がないケースは全サイクルで同一。
- 確率系(GV-22〜24)は注入乱数で検証する。
- **本表の全ケースを単体テストとして実装し、全件パスすること。**

## 10. パーサ実装への要求(機能仕様書 §3.1 の再掲+補足)

1. `parse(code: string) → { ast } | { error: { code, line, message } }` の純粋関数。
2. `query(ast, cycleIndex, rng) → Event[]` の純粋関数。同一入力(+同一乱数列)で同一出力(参照透過)。
3. DOM / Tone.js / タイマー非依存。Vitest で Node 実行できること。
4. ゴールデンテストベクタのテストコードは `tests/notation.golden.test.ts` に、本書の GV-ID をテスト名に含めて実装すること(検収時の突合のため)。
