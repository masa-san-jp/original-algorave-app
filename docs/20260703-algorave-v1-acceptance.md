## 5. 自動テストの必須範囲(区分A の詳細)

1. 記法仕様書 §8 の GV-01〜GV-38(GV-11b 含む)全件。テスト名に GV-ID を含める。
2. Bjorklund 規範ベクタ(記法仕様書 §5.1)。
3. `query` の参照透過性: 同一 AST・同一サイクル・同一モック乱数で2回評価し同一結果。
4. parser がブラウザ API 非依存で Node 上で走ること(CI 上で担保される)。
5. カバレッジ: `parser` / `pattern` モジュールで branch 90% 以上。
